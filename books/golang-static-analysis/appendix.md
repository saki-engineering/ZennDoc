# おわりに
というわけで、Goでの静的解析ツール実装のやり方について見てきましたが、いかがでしたでしょうか。

Goは「静的解析がやりやすいように設計しよう」という背景を持って作られたプログラミング言語なので、`go/ast`や`go/types`といった解析用の標準ライブラリがとても充実しています。
また、`skeleton`のような有志による解析ツール用の実装フレームワークも作られており、Goは静的解析ツールを作る上でとても整ったエコシステムを持つと言えるでしょう。

しかし、ライブラリやフレームワークがあっても、結局その使い方が理解できないのであれば解析ロジックを書くことはできません。
- `skeleton`を使ってディレクトリ構造をいい感じに生成してみたけど、解析対象のASTや型情報がどこに入っているんだろう？
- `ast.Node`や`types.Object`ってよく登場するけど、これは何者？
- `types.Implements`を使えばやりたい処理が書けそうだけど、引数に与えるための`types.Type`型をどうやってASTから作ればいいのかわからない……

筆者が`slog.Handler`のLTを作ろうとしたときに初めて静的解析ツール作成に手を出したのですが、この辺りの知識がなくてとても苦労しました。
`go/ast`や`go/types`のパッケージそのものの説明はインターネット上にいくつか存在するのですが、「`skeleton`を用いて生成した`analysis.Analyzer`の形式で解析ロジックを書き始めたときに、どうやって`go/ast`や`go/types`に用意されている関数やメソッドを使えるように値をこねくり回すか」という視点を解説している記事はないのではないかと感じました。

この本は「静的解析ツールを初めて作り始めたけど、`skeleton`で雛形を生成してから先が全くわからない」という方が対面するであろう上記のつらみを軽減できるように執筆しました。
読んだ方が「`go/ast`や`go/types`の全容までは把握しきれていないけど、解析対象のプログラムファイルを読み込んで、上記パッケージを使って自分のやりたい処理を書くイメージが付いた」という状態になってもらえたらとても嬉しいです。

また、今回は触れませんでしたが、静的解析の技術はASTの構文解析や型チェックだけではなく、SSAやポインタ解析など他にも色々存在します。
この本で学んだ知識を土台にして、より高度なそれらの解析にチャレンジしてみるのも面白いのではないでしょうか。
